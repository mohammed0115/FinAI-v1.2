# تقرير تدقيق المستند الشامل
# Comprehensive Invoice Audit Report

## نظرة عامة
يوفر النظام الآن تقرير تدقيق شامل لكل فاتورة مع جميع البيانات المستخرجة والتحليلات.

## المكونات الرئيسية

### 1. معلومات سجل المستند
- معرف المستند (Document ID)
- تاريخ التحميل (Upload Date)
- محرك OCR المستخدم
- درجة ثقة OCR (0-100%)
- حالة المعالجة

### 2. بيانات الفاتورة المستخرجة
#### معلومات الفاتورة:
- **رقم الفاتورة** (Invoice Number)
- **تاريخ الإصدار** (Issue Date)
- **تاريخ الاستحقاق** (Due Date)
- **العملة** (Currency)

#### بيانات الموردة (Vendor):
- اسم الموردة
- عنوان الموردة

#### بيانات المشتري (Customer):
- اسم المشتري
- عنوان المشتري
- رقم التعريف الضريبي (TIN)

#### تفاصيل البنود (Line Items):
جدول يظهر:
- اسم المنتج
- الوصف
- الكمية
- سعر الوحدة
- الخصم
- الإجمالي

#### الإجماليات:
- المجموع الجزئي
- الضريبة (VAT)
- المبلغ الإجمالي

### 3. نتائج التحقق (Validation Results)

يتم التحقق من:
- ✔ تم اكتشاف رقم الفاتورة
- ✔ معلومات الموردة
- ✔ معلومات المشتري
- ✔ الأصناف المكتشفة
- ✔ توافق إجمالي الفاتورة
- ⚠ معلومات الضريبة (VAT)

### 4. نتائج الامتثال (Compliance Checks)

يتم فحص:
- وجود رقم التعريف الضريبي
- صحة تنسيق الفاتورة
- صحة صيغة العملة
- بيانات الضريبة

### 5. كشف التكرار (Duplicate Detection)
- درجة التكرار (0-1)
- المستندات المطابقة
- الحالة: متطابق أم لا

### 6. كشف الأنماط غير الطبيعية (Anomaly Detection)
- درجة الشذوذ (0-1)
- الحالة: عادي أم غير عادي
- شرح الشذوذ

### 7. تقييم المخاطر (Risk Score)

#### مستويات المخاطر مع التعليقات النصية:

**🟢 منخفضة (Low)**
- الدرجة: 0-25
- الحالة: مستوى منخفض من المخاطر
- الشرح: الفاتورة تبدو صحيحة وموثوقة. لا توجد مشاكل جوهرية.
- التوصية: موافقة الفاتورة

**🟡 متوسطة (Medium)**
- الدرجة: 26-50
- الحالة: مستوى متوسط من المخاطر
- الشرح: هناك بعض المشاكل التي تحتاج إلى مراجعة إضافية قبل الموافقة.
- التوصية: مراجعة وتحقق إضافي

**🟠 عالية (High)**
- الدرجة: 51-75
- الحالة: مستوى عالي من المخاطر
- الشرح: هناك عدة مشاكل تتطلب تدخل يدوي فوري للمراجعة والموافقة.
- التوصية: تدخل يدوي مطلوب

**🔴 حرجة (Critical)**
- الدرجة: 76-100
- الحالة: مستوى حرج من المخاطر
- الشرح: الفاتورة فيها مشاكل خطيرة جداً. يجب رفضها أو تصحيحها بشكل كامل.
- التوصية: رفض أو تصحيح كامل

### 8. ملخص الذكاء الاصطناعي (AI Summary)
- **الملخص التنفيذي**: تحليل شامل من قبل الذكاء الاصطناعي
- **المخاطر الرئيسية**: قائمة بأهم المخاطر المكتشفة
- **الإجراءات الموصى بها**: خطوات العمل المقترحة
- **الحالة النهائية**: موافقة/مراجعة/رفض

### 9. التوصيات النهائية
- موافقة الفاتورة
- التحقق من معلومات الضريبة
- طلب تصحيحات

### 10. سجل التدقيق (Audit Trail)
إظهار جميع الأحداث بالتسلسل الزمني:
- تحميل المستند
- معالجة OCR
- استخراج البيانات
- فحص الامتثال
- إنشاء التقرير

### 11. حالة المراجعة (Review Status)
- حالة المراجعة (انتظار/تمت/مرفوضة)
- المراجع المسؤول

## الوصول إلى التقرير

### من واجهة المستخدم:
1. اذهب إلى **تقارير التدقيق الذكية** من القائمة الجانبية
2. انقر على **تقرير** للفاتورة المرغوبة
3. اعرض التقرير الشامل

### من الكود:
```python
# في views.py
invoice_audit_report_view(request, extracted_data_id)

# في URLs
path('reports/audit/<uuid:extracted_data_id>/', invoice_audit_report_view, name='invoice_audit_report')
```

## طريقة العمل

### المرحلة 1: استخراج البيانات
عند رفع مستند:
1. معالجة OCR (Tesseract أو OpenAI Vision)
2. استخراج البيانات البنيوية
3. حفظ في OCREvidence

### المرحلة 2: معالجة البيانات
1. تطبيع البيانات
2. التحقق من الصحة
3. إنشاء ExtractedData

### المرحلة 3: فحص الامتثال
1. التحقق من صيغة الفاتورة
2. فحص بيانات الضريبة
3. كشف التكرار
4. كشف الشذوذ

### المرحلة 4: تقييم المخاطر
1. حساب درجة المخاطر (0-100)
2. تحديد مستوى المخاطر
3. إضافة التعليقات النصية

### المرحلة 5: تحليل الذكاء الاصطناعي
1. جمع نتائج الفحص
2. استدعاء OpenAI API
3. إنشاء ملخص شامل

### المرحلة 6: إنشاء التقرير
1. تجميع كل البيانات
2. عرض التقرير في الويب
3. إمكانية الطباعة والتصدير

## ملاحظات مهمة

1. **جودة البيانات**: تعتمد جودة التقرير على دقة استخراج OCR
2. **معدل الثقة**: يتم عرض معدل ثقة OCR لكل فاتورة
3. **البيانات اليدوية**: يمكن للمستخدمين تصحيح البيانات يدويًا
4. **تخزين البيانات**: يتم حفظ جميع البيانات في قاعدة البيانات
5. **السجل التدقيقي**: يتم تسجيل جميع العمليات للمراجعة لاحقًا
